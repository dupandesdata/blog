JavaScript Type Number

Angka pada javascript tidak ada bilangan integer atau bilangan bulat yang ada hanya floating point atau angka pecahan. Angka di dalam Javascript terdiri dari 64bit / 2⁶⁴ / 18x10¹⁸ / 18x1milyar milyar, angka pada JavaScript memiliki standar internasional berikut ini beberapa standar internasional : 

  • IEEE 754-1985 (Binary Floating-Point Arithmetic) standar yang diterbitkan pada tahun 1985. Standar ini hanya mencakup aritmatika titik-mengambang biner.
  • IEEE 754-2008 (Floating-Point Radix-Independen) diterbitkan pada Agustus 2008, standar tersebut mengacu pada standar IEEE 754-1985 asli.
  • IEEE 754-2019, diterbitkan pada Juli 2019. Standar ini adalah revisi kecil dari dan mengantikan IEEE 754-2008, IEEE 754-2019 ini mencakup klarifikasi seperti totalOrder, perbaikan seperti minNum dan penambahan operasi baru hasil rekomendasi seperti augmentedAddition. 

Untuk saat standar angka yang digunakan oleh JavaScript mengikuti standar IEEE 754-2019 dan akan diperbarui pada projects 2028.  Sumber Wikipedia.

Bilangan Integer 

Syntax JavaScript 

10, 2500 dan 123456

Loh kan tadi katanya javascript tidak ada bilangan integer? Ia betul tapi jika kita menemukan bilangan integer pada javascript akan diproses secara pecahan tapi hanya akurat sampai 15 digit.

Studi kasus :

Kita tuliskan angka lalu enter javascript akan mengembalikan nilai tersebut.

Console

> 20

< 20

> 2500

< 2500

> 123456

< 123456

> 999999999999999

< 999999999999999

Penjelasan:

  • 20 lalu enter hasilnya 20
  • 2500 lalu enter hasilnya 2500
  • 123456 lalu enter hasilnya 123456
  • 999999999999999 lalu enter hasilnya sama saja
  •  jika kita menuliskan angka 9 sebanyak 16 digit apa yang terjadi? 

Console

> 9999999999999999

< 1000000000000000

Penjelasan:

Jika kita kirimkan 16 digit keatas maka JavaScript akan memproses secara pecahan dan angka tersebut akan dibulatkan untuk menjadi angka yang berbeda bisa keatas atau kebawah. Jika kita ingin membuat 16 digit keatas maka disarankan untuk menggunakan bilangan BigInt. 

Bilangan Biglnt 

Apa itu BigInt? Biglnt adalah sebuah bilangan bulat dengan presisi arbitrer. Dengan menggunakan bilangan biglnt kita dapat meyimpan angka 16 digit keatas agar tidak dapat dibulatkan seperti studi kasus di atas, lalu bagaimana cara penulisan nya? Untuk membuat bilangan biglnt kita hanya menuliskan n diakhiri angka contoh :

12345678901234569n // 17 digit 

hasilnya tidak akan dibulatkan. 

Console

> 12345678901234569n

< 12345678901234569n

Kita juga dapat menyimpan angka desimal tetapi koma diganti dengan titik, hati-hati saat menulis!.

Syntax JavaScript 

0.3 

4.15

200.123456789 

Kita hanya bisa menulis 17 digit dibelakang titik hati-hati saat menjumlahkan bilangan pecahan.

Console

> 0.3 + 0.7

< 1

> 0.1 + 0.2

< 0.30000000000004

Penjelasan:

0.3 + 0.7 = 1 tidak ada yang aneh, namun jika kita menjumlahkan 0.1 + 0.2 = 0.30000000000004 mengapa ini terjadi? Karena sebenarnya JavaScript menjumlahkan secara pecahan dan ada pecahan yang cukup kecil jadi hasilnya tidak bulat.

Maka hati-hati saat menjumlahkan bilangan pecahan karena hasilnya tidak seperti yang kita inginkan. 

Bilangan Eksponen

Kita juga dapat menyimpan bilangan yang sangat besar dan sangat kecil dengan eksponen.

Console 

> 123e5 

< 12300000

> 123e-5 

< 0.00123 

Kita juga dapat menyimpan angka minus seperti -10 dan -0.4 JavaScript akan mengirimkan kembali nilai tersebut. 

Console 

> -10

< -10

> -0.4

< 0.4

Jadi kesimpulannya didalam 64bit tadi tidak dapat digunakan untuk membuat bilangan bulat (positif) saja tetapi banyak yang bisa kita gunakan, berikut adalah urutan penggunaan bit :

64bit itu terbagi menjadi tiga bagian seperti gambar di atas.

  • 1bit untuk Sign bit bilangan negatif (minus)
  • 11bit untuk Exponent bilangan positif (bulat)
  • 53bit untuk Significand precision disimpan secara eksplisit untuk bilangan sebenarnya termasuk pecahan

Hati-hati jika kita menuliskan angka pada javascript jangan awali dengan 0 memang kenapa? Contoh kita cari tahu

Console

> 022

< 18

Kenapa kok jadi 18? Karena bilangan yang diawali dengan 0 dianggap sebagai bilangan oktal, bilangan yang berbasis 8. Apa itu pengertian dari bilangan oktal? Bilangan hasil pengelompokan dari 3bit bilangan biner yang diurutkan dari belakang.

Hati-hati saat menuliskan angka pada javascript dengan diawali oleh 0x javascript akan mengembalikan nilai tersebut dalam bentuk bilangan hexadesimal. Apa itu bilang hexadesimal? Bilangan yang berbasis 16 yaitu 123456789ABCDEF.

JavaScript juga memiliki angka spesial yaitu:

  • Infinity
  • -Infinity 
  • NaN (Not a Number)

Infinity adalah angka spesial dengan properti khusus yang disebut Infinity positif dan Infinity negatif.

Positif Infinity

> 10 / 0

< Infinity

Penjelasan:

Nilai positif apa pun yang dibagi dengan 0 maka hasilnya adalah positif Infinity.

Negatif Infinity 

> -2 / 0

< -Infinity

Penjelasan:

Nilai negatif apa pun yang dibagi dengan 0 maka hasilnya adalah negatif Infinity.

Hati-hati jika mengurang, mengali & membagi sebuah bilangan dengan string! Hasilnya NaN (Not a Number).

Console

> 10 - "jeruk"

< NaN

> 10 * "jeruk"

< NaN

> 10 / "jeruk"

< NaN

> 0 / 0

< NaN

Hati-hati jika mengurang, mengali & membagi angka dengan string, string tersebut dianggap sebagai bilangan integer, jadi javascript akan mengurang, mengali & membagi sebagai mana mestinya.

Console

> 10 - "5"

< 5

> 100 / 10

< 10 

> 10 * 10

< 100
